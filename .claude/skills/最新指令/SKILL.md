# 最新指令管理系统

## 概述

此 skill 用于管理交易员的临时指令和任务. AI 代理必须:

1. **每次对话开始时读取** `docs/最新指令.md`
2. **遵守有效指令** (检查状态和类型)
3. **及时归档已完成任务** 到 `docs/历史指令/` 目录
4. **保持最新指令文件干净** - 只保留有效任务

## 文件结构

```
docs/
├── 最新指令.md                    # 当前有效指令 (每次对话必读)
└── 历史指令/                      # 已归档的任务
    ├── 2025-12-20_降息做空.md
    ├── 2025-12-19_BOLL交易.md
    └── ...
```

## 指令类型

### 1. 全局禁令 (Global Restriction)

- **有时效性的约束条件**, 在有效期内所有交易都必须遵守
- 优先级最高, 违反禁令的任务直接拒绝执行
- 过期后自动失效, 无需手动清理 (但建议清理保持干净)
- 示例:
  - "今天不要做空" → 有效期: 2025-12-20 23:59
  - "这周不要交易 ETH" → 有效期: 2025-12-22 23:59
  - "永远不要做空 BTC" → 有效期: 永久

**时效规则:**
| 表述 | 有效期 |
|-----|-------|
| 今天 | 当日 23:59 |
| 这周 | 本周日 23:59 |
| 这个月 | 本月最后一天 23:59 |
| 永久/一直 | 永久 |
| 未指定 | 默认当日 23:59 |

### 2. 一次性任务 (One-time Task)

- 触发条件满足后执行一次即完成
- 完成后立即归档到 `docs/历史指令/日期_摘要.md`
- 示例: "美联储降息消息出来就做空"

### 3. 周期性任务 (Recurring Task)

- 持续有效, 可多次触发执行
- 交易员明确说"结束/停止/取消"才归档
- 示例: "BOLL顶部做空底部做多"

## 指令格式规范

### docs/最新指令.md 格式

```markdown
# 最新指令

> 最后更新: YYYY-MM-DD HH:MM

## 全局禁令

- [ ] 禁令内容 | 有效期: YYYY-MM-DD HH:MM | 原因: xxx

示例:
- [ ] 今天禁止做空 | 有效期: 2025-12-20 23:59 | 原因: 市场不确定
- [ ] 禁止交易 DOGE | 有效期: 永久 | 原因: 波动太大

## 周期性任务

- 任务ID: C001 | 类型: recurring | 创建: YYYY-MM-DD HH:MM
  - 指令: 具体指令内容
  - 币种: BTC/ETH/...
  - 时间周期: 15m (默认)
  - 触发条件: ...
  - 执行策略: ...
  - 止盈止损: ...
  - 执行记录:
    - YYYY-MM-DD HH:MM: 做空 BTC @ 100000, 盈利 +50 USDT
    - YYYY-MM-DD HH:MM: 做多 ETH @ 3500, 亏损 -20 USDT

## 一次性任务

- 任务ID: T001 | 类型: one-time | 状态: pending | 创建: YYYY-MM-DD HH:MM
  - 指令: 具体指令内容
  - 触发条件: ...
  - 执行策略: ...
```

### docs/历史指令/日期_摘要.md 格式

```markdown
# 任务名称

- 任务ID: T001 / C001
- 类型: one-time / recurring
- 创建时间: YYYY-MM-DD HH:MM
- 结束时间: YYYY-MM-DD HH:MM
- 结束原因: 任务完成 / 交易员取消 / 条件过期

## 原始指令

交易员的原始指令内容

## 执行记录

| 时间 | 操作 | 币种 | 方向 | 入场价 | 出场价 | 盈亏 |
|-----|------|-----|------|-------|-------|------|
| 2025-12-20 10:00 | 开仓 | BTC | 空 | 100000 | - | - |
| 2025-12-20 10:30 | 平仓 | BTC | 空 | - | 99500 | +50 USDT |

## 行情回顾

任务期间的市场走势概述

## 总结

- 总交易次数: X
- 盈利次数: X
- 亏损次数: X
- 总盈亏: +/- XXX USDT
- 经验教训: ...
```

## AI 代理行为规范

### 对话开始时

1. 读取 `docs/最新指令.md`
2. **检查全局禁令** - 有生效中的禁令必须告知交易员
3. **清理过期禁令** - 已过期的禁令可以删除
4. 汇报当前有效任务
5. 如有周期性任务, 检查是否有触发条件满足

### 收到新指令时

1. **判断指令类型**:
   - 是约束/禁止类 (如"今天不要做空") → 全局禁令
   - 有明确触发事件 (如"降息消息出来") → 一次性任务
   - 是持续策略 (如"BOLL顶底交易") → 周期性任务
2. **不确定的信息必须追问**:
   - 币种? (默认询问)
   - 时间周期? (默认 15m)
   - 具体触发条件?
   - 止盈止损规则?
   - 仓位大小?
3. **确认清楚后才记录到** `docs/最新指令.md`

### 任务完成/结束时

1. **一次性任务完成**:
   - 记录执行结果、行情走势、盈亏
   - 归档到 `docs/历史指令/YYYY-MM-DD_摘要.md`
   - 从 `docs/最新指令.md` 中删除

2. **周期性任务结束** (交易员明确说结束):
   - 汇总所有执行记录
   - 记录行情回顾和总结
   - 归档到 `docs/历史指令/YYYY-MM-DD_摘要.md`
   - 从 `docs/最新指令.md` 中删除

### 默认值

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| 时间周期 | 15m | K线周期 |
| 仓位管理 | 固定风险法 (1R = 2%) | 参考固定风险仓位管理 skill |

### 追问示例

交易员: "BOLL 顶部做空底部做多"

AI 应追问:
1. 交易哪个币种? (BTC/ETH/其他)
2. 时间周期用 15 分钟可以吗?
3. 触及上下轨直接入场, 还是等反转信号?
4. 止损放在轨道外多少?
5. 止盈目标是中轨还是固定盈亏比?

## 注意事项

1. **不确定就追问** - 交易规则必须明确才执行
2. **归档要完整** - 记录行情、盈亏、经验教训
3. **保持最新指令干净** - 已完成的及时清理
4. **周期任务持续有效** - 除非交易员说停止
5. **全局禁令优先级最高** - 任何任务都必须先检查是否违反禁令
6. **禁令有时效性** - 过期自动失效, 但交易时仍需检查
