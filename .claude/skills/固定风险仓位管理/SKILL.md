# 固定风险金额仓位管理法 (Fixed Risk Position Sizing)

又称: R-Based Position Sizing / 固定R值法

## 核心概念

### 什么是 R

- **R** = 单笔交易的风险金额 (Risk per Trade)
- **标准设定: 账户总资金的 2%**
- 例: 10,000 USD 账户, 2% 风险 = 1R = 200 USD

### 计算公式

```
仓位大小 = 风险金额 / 止损点数
         = (账户资金 × 风险比例) / (入场价 - 止损价)
         = R / 止损距离
```

## 优势

1. **固定最大亏损**: 每笔交易亏损固定, 不会因仓位过大爆仓
2. **自动调节仓位**: 止损近 → 仓位大 → 盈利潜力大; 止损远 → 仓位小 → 保护资金
3. **盈亏比优化**: 在关键位置(顶/底)入场, 自然获得更好的风险回报比
4. **心理稳定**: 每次最多亏 1R, 减少情绪波动

## 实战示例

### 账户条件

- 账户资金: 10,000 USD
- 单笔风险: 2% = 200 USD (1R)
- 交易品种: BTC/USDT

### 场景一: 顶部做空 (小止损, 大仓位)

```
识别顶部: 30,000 USD
止损位置: 30,300 USD (顶部上方 1%)
止损距离: 300 USD (1%)

计算仓位:
仓位价值 = 200 / 0.01 = 20,000 USD
BTC 数量 = 20,000 / 30,000 = 0.667 BTC

如果跌到 27,000:
盈利 = 0.667 × 3,000 = 2,000 USD = 10R
```

### 场景二: 中间做空 (中等止损, 中等仓位)

```
入场位置: 28,500 USD
止损位置: 30,000 USD (顶部)
止损距离: 1,500 USD (5.26%)

计算仓位:
仓位价值 = 200 / 0.0526 = 3,802 USD
BTC 数量 = 3,802 / 28,500 = 0.133 BTC

如果跌到 27,000:
盈利 = 0.133 × 1,500 = 200 USD = 1R
```

### 场景三: 底部附近做空 (大止损, 小仓位)

```
入场位置: 27,500 USD
止损位置: 30,000 USD (顶部)
止损距离: 2,500 USD (9.09%)

计算仓位:
仓位价值 = 200 / 0.0909 = 2,200 USD
BTC 数量 = 2,200 / 27,500 = 0.08 BTC

如果跌到 27,000:
盈利 = 0.08 × 500 = 40 USD = 0.2R
```

## 关键结论

| 入场位置 | 止损距离 | 仓位大小 | 同样下跌盈利 | 风险回报比 |
|---------|---------|---------|------------|-----------|
| 顶部     | 小      | 大      | 高         | 优秀      |
| 中间     | 中      | 中      | 中         | 一般      |
| 底部     | 大      | 小      | 低         | 差        |

**在顶部入场做空的优势**:
- 止损距离小 → 仓位自动放大
- 下跌空间大 → 盈利自动放大
- 固定风险 → 潜在收益最大化

## 实施要点

1. **严格计算**: 入场前必须计算好仓位, 不可凭感觉
2. **止损必设**: R值的前提是止损必须执行
3. **记录R值**: 用 R 来衡量盈亏, 例: +3R, -1R
4. **期望值追踪**: 胜率 × 平均盈利R - 败率 × 1R > 0

## 注意事项

1. **杠杆限制**: 仓位价值可能超过账户余额, 需要使用杠杆或降低风险比例
2. **滑点考虑**: 实际止损可能有滑点, 建议止损距离留有余量
3. **手续费**: 开仓/平仓手续费会侵蚀利润, 计算时应考虑
4. **最小下单量**: 交易所有最小下单限制, 仓位过小可能无法开仓
